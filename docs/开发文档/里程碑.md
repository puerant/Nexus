## M1 · 项目骨架 & 核心基础设施

**目标**：搭建可运行的 Tauri 应用框架，实现 Workspace 管理和基础导航，为后续功能模块提供稳固基础。

**计划周期**：2026-03-16 ~ 2026-03-29（2 周）

### 任务清单

#### 1.1 工程搭建
- [x] 初始化 Tauri 2.x + Vue 3 + TypeScript 项目
- [x] 配置 ESLint / Prettier / TypeScript 严格模式
- [x] 配置 Vite 构建与热更新
- [x] 建立项目目录结构（`src/views`、`src/components`、`src/stores`、`src-tauri/src`）

#### 1.2 主界面布局
- [x] 实现三栏布局：顶部导航栏 / 左侧边栏 / 主工作区
- [x] 侧边栏：项目列表、新建项目入口、导航图标
- [x] 顶部导航栏：Logo、全局设置入口、模型配置入口
- [x] 阶段 Tab 导航（五阶段，支持柔性跳转）
- [x] 响应对应原型页面 `03-project.html` 的整体布局

#### 1.3 Workspace 管理（Rust 后端）
- [x] Tauri 命令：打开本地目录（`open_workspace`）
- [x] Tauri 命令：扫描 Workspace 目录结构（`scan_workspace`）
- [x] Tauri 命令：读写文件（`read_file` / `write_file`）
- [x] 安全约束：文件操作严格限制在 Workspace 根目录内
- [x] Workspace 选择页面（对应原型 `01-workspace.html`）：近期记录列表 + 拖拽/选择目录

#### 1.4 配置同步（可选启动项）
- [x] 配置同步向导页（对应原型 `02-sync.html`）：GitHub 连接 / 跳过选项
- [x] 同步配置写入 `project.json`

#### 1.5 项目管理基础
- [x] 新建项目流程：初始化 `.ai-workflow/` 目录结构
- [x] `project.json` 读写：项目名称、五阶段状态、AI 模型配置

## M2 · 需求梳理模块

**目标**：实现 AI 驱动的需求收集与结构化，输出可锁定的 Markdown 需求文档和功能思维导图。

**计划周期**：2026-03-30 ~ 2026-04-19（3 周）

### 任务清单

#### 2.1 Chat 对话界面
- [ ] 消息列表：用户消息 / AI 消息气泡
- [ ] 流式输出渲染（Anthropic TS SDK `stream`）
- [ ] 输入框：多行文本、发送快捷键
- [ ] 对话历史持久化到 `.ai-workflow/requirements/` 目录

#### 2.2 AI 主动追问流程
- [ ] System Prompt 模板：需求分析师角色，启用 AskUserQuestion 模式
- [ ] 追问维度覆盖：功能边界、用户角色、优先级、技术约束、排除项
- [ ] 多轮对话后自动生成需求草稿（结构化 Markdown）

#### 2.3 需求文档编辑器
- [ ] Markdown 编辑器（双栏：编辑 + 预览）
- [ ] 提示词面板侧边栏（对应原型 `04-requirements.html`）
- [ ] 类终端输出区：AI 流式生成过程展示

#### 2.4 功能思维导图
- [ ] 集成 AntV G6
- [ ] AI 输出 Markdown 层级结构 → 自动转换为 G6 图数据并渲染
- [ ] 思维导图与需求文档双向关联

#### 2.5 锁定需求操作
- [ ] "锁定需求"按钮：写入锁定状态标记至需求文档头部
- [ ] 锁定后进入只读模式，提供"解锁修改"入口
- [ ] 锁定事件记录到项目活动日志

**交付标准**：
- 可与 AI 多轮对话并获得追问
- 可生成结构化需求文档（Markdown，保存至 Workspace）
- 可渲染功能思维导图
- 需求文档可锁定 / 解锁

## M4 · 原型图设计模块

**目标**：AI 读取需求文档和技术选型文档，自动生成可交互 HTML 原型，支持迭代修改和手动版本存档。

**计划周期**：2026-05-04 ~ 2026-05-10（1 周）

### 任务清单

#### 4.1 原型生成
- [ ] AI 读取锁定需求文档和技术选型文档，按页面逐一生成 HTML 原型
- [ ] 流式输出进度展示（类终端）
- [ ] 将生成的 HTML 文件写入 `.ai-workflow/prototypes/` 目录（Rust 后端）

#### 4.2 文件树 & 预览
- [ ] 左侧文件树面板：展示 prototypes 目录下所有原型文件（对应原型 `05-prototype.html`）
- [ ] 右侧 iframe 预览：加载选中的 HTML 文件，支持交互
- [ ] 文件切换、重命名

#### 4.3 原型迭代
- [ ] 描述修改意图输入框 + 发送
- [ ] AI 读取当前原型文件内容 + 修改意图 → 生成新版本
- [ ] 流式输出渲染修改过程

#### 4.4 版本管理
- [ ] 手动打标签存档：用户输入版本名（如"v1-初始版"）
- [ ] 存档版本列表：仅展示手动存档的版本
- [ ] 版本回退：将存档版本内容恢复为当前工作版本

**交付标准**：
- AI 可根据需求文档和技术选型生成多页 HTML 原型
- iframe 内原型可正常交互
- 手动存档版本可创建、查看、回退

## M5 · 任务开发模块

**目标**：实现完整的 AI 辅助任务拆解与自动执行流程，含任务看板、流式执行、Code Review、提示词管理。

**计划周期**：2026-05-11 ~ 2026-06-14（5 周）

### 任务清单

#### 5.1 任务层级与数据结构
- [ ] 定义 Milestone / Module / Task 数据结构（`milestones.json` / `task.json`）
- [ ] Tauri 命令：CRUD 任务、里程碑、模块
- [ ] 任务状态机：待执行 → 执行中 → 待验收 → 已完成

#### 5.2 任务看板界面（对应原型 `06-tasks.html`）
- [ ] 看板视图（Kanban）：四列状态泳道，拖拽修改状态
- [ ] 列表视图：任务表格，可排序筛选
- [ ] 视图切换按钮
- [ ] 任务详情抽屉/弹窗：名称、描述、验收用例、提示词、Review 结果
- [ ] 左侧面板：活跃度热力图 + 任务统计（按类型/状态）

#### 5.3 AI 任务拆解
- [ ] 输入：需求文档 + 技术选型文档
- [ ] AI 输出：里程碑列表 → 模块列表 → 任务列表（含验收用例）
- [ ] 拆解结果可人工编辑调整

#### 5.4 AI 自动执行引擎
- [ ] 里程碑级"AI 自动执行"触发器
- [ ] 任务独立上下文窗口（每任务开启新对话）
- [ ] 上下文构建：记忆模块检索（标签过滤）+ 任务信息注入 + System Prompt 组装
- [ ] 类终端流式输出面板：AI 思考过程实时展示
- [ ] 暂停等待人工确认场景：技术方案分支选择、路径不确定、依赖未完成、低置信度
- [ ] 代码生成 → invoke Rust 后端 → 写入 Workspace 项目文件

#### 5.5 记忆模块
- [ ] 数据结构：`prompts.json` / `test-cases.json` / `task-templates.json` / `experiences.json`
- [ ] 基于标签的检索（技术栈标签 + 业务领域标签 + 任务类型标签）
- [ ] 读写接口：Tauri 命令封装

#### 5.6 AI Code Review
- [ ] 任务执行完成后自动触发 Code Review
- [ ] Review 维度：代码质量、安全漏洞、性能问题、验收用例符合度
- [ ] Review 结果写入 `task/<task-id>/review.md`

#### 5.7 提示词管理
- [ ] 预设提示词模板库（含阶段/任务类型分类）
- [ ] 执行前提示词编辑器（微调）
- [ ] 任务完成后 1-5 星人工评分
- [ ] 评分 + 使用次数 → 推荐权重计算

**交付标准**：
- AI 可将需求拆解为三层任务结构
- 可触发里程碑级 AI 自动执行，流式输出任务进度
- 代码可写入 Workspace 本地文件
- Code Review 自动生成并保存
- 提示词可评分，评分影响推荐排序

## M6 · 项目复盘模块

**目标**：项目结束后自动聚合数据，AI 生成复盘报告，经验沉淀到四个复用库。

**计划周期**：2026-06-15 ~ 2026-06-28（2 周）

### 任务清单

#### 6.1 数据聚合
- [ ] 自动收集：所有任务执行记录、提示词使用记录、测试用例、阻塞点
- [ ] 统计计算：里程碑完成率、任务通过率、提示词平均评分

#### 6.2 复盘报告生成（对应原型 `07-review.html`）
- [ ] AI 生成复盘报告（Markdown），包含：里程碑完成情况、技术决策回顾、最优提示词、改进建议
- [ ] 报告页面：汇总统计卡片、提示词效果排行榜、阶段完成情况对比、全项目热力图

#### 6.3 经验沉淀到复用库
- [ ] 高评分提示词 → 写入 `memory/prompts.json`（去重 + 合并）
- [ ] 验收通过的测试用例 → 写入 `memory/test-cases.json`
- [ ] 成功的任务拆解方案 → 写入 `memory/task-templates.json`
- [ ] 项目经验摘要（AI 生成）→ 写入 `memory/experiences.json`，附技术栈/业务领域标签

**交付标准**：
- 复盘报告可自动生成并保存
- 四个复用库数据更新后，在新项目任务执行时可检索到

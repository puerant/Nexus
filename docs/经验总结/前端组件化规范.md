### 推荐的组件目录结构 
   在你的项目中，建议采用以下三层组件规范：

#### 第一层：Base/Atom (基础原子组件)
内容：Button, Input, Checkbox, Icon, Tag。

规范：不包含任何业务逻辑，只负责样式和基础交互。必须高度可配置（通过 Props）。

路径：src/components/base/

#### 第二层：Business/Molecule (业务单元组件)
内容：TaskItem, SearchBar, UserAvatar, ProjectCard。

规范：由原子组件组合而成，包含特定的业务数据结构，但不直接发起 API 请求（通过 Emit 传递事件）。

路径：src/components/business/

#### 第三层：Layout/Organism (布局/模块组件)
内容：Sidebar, KanbanBoard, DetailPanel。

规范：负责复杂的布局控制，管理子组件的排列。

路径：src/layout/ 或 src/components/layout/

### 组件化的“五大规范”
   为了减少注意力分散，你的组件化规范应包含：

#### A. 间距规范 (Spacing Tokens)
禁止在组件内写 margin: 13px 这样的死数字。

规定：所有间距必须是 4px 或 8px 的倍数。

示例：p-4 (16px), m-2 (8px)。

#### B. 命名规范 (Naming Convention)
文件命名：始终使用大驼峰（PascalCase），如 TaskCard.vue。

Prop 命名：在模板中使用 kebab-case，在脚本中使用 camelCase。

私有前缀：如果是项目特有的基础组件，建议加前缀（如 AppButton.vue），防止与第三方库冲突。

#### C. 状态驱动规范 (State-Driven)
规范：组件的所有视觉变化（Loading, Disabled, Active）必须通过 Props 驱动，而不是直接操作 DOM。

#### D. 单向数据流 (Data Flow)
规范：Props Down, Events Up。子组件绝不直接修改父组件传下来的 Ref 或 Reactive 对象。

#### E. 插槽规范 (Slots First)
规范：为了保持灵活性，优先使用 Slot 而不是过多的 Props 来传递复杂的 HTML 内容。
